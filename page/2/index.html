使用CocoaPods管理开源框架及安装详细步骤

2015-06-23

众所周知，我们在进行项目开发时,经常会使用到成熟的第三方框架,手动拖入到工程不仅会增加包的大小,而且不利于及时更新。CocoaPods可以很好的解决这两方面的缺陷,以下是安装CocoPods的详细步骤:

一、修改为Ruby淘宝路径

1.查看Ruby环境（）
ruby –version
附:
升级Ruby环境，首先需要安装rvm
1> curl -L get.rvm.io | bash -s stable
2> source ~/.bashrc
3> source ~/.bash_profile
4> 查看rvm版本
rvm -v
(1.29.3 (latest))
5> 列出ruby可安装的版本
rvm list known
6> 安装一个ruby版本
rvm install 2.4.1

2.移除本来的Ruby路径
gem sources –remove https://rubygems.org/

3.添加淘宝镜像
gem sources –a https://ruby.taobao.org (此处需要注意的是iOS9之前路径为 http://ruby.taobao.org iOS9之后需要修改)

4.检查镜像是否正确
gem sources –l

二、安装CocoaPods

1.输入以下命令后,输入管理员密码,等待安装。
sudo gem install cocoapods

三、创建Podfile文件

1.进入到项目目录下 cd 项目目录

2.初始化CocoaPods环境(会自动创建Podfile文件)
pod init

3.然后编辑Podfile文件
vim Podfile
(注意：在文件中写入需要导入的库，以及三方库的版本 如果不写版本，会默认最新版
)
按下i键，进入编辑状态

然后在target do与end之间写入需要导入的库

例如：pod ‘AFNetworking’, ‘~> 2.1.0’

5.输入完以后，按下Esc键退出编辑状态，然后输入 :wq 命令，保存并退出

四、通过Pod导入库

1.在导入以前，最好先输入以下命令，重新检测下新的三方库。在以后需要更新的时候，也最好先输入下面这个命令
pod setup

2.输入以下命令开始导入
pod install

3.打开工程并使用
以后打开工程的时候，去打开project.xcworkspace
如下方式使用框架： #import

4.更新工程中的pod
比如现在需要导入一个新的库SDWebImage，那么需要修改Podfile。
输入命令
vim Podfile
同之前编辑Podfile步骤一样

输入下面命令，更新Pod。如果很久没更新过了，先输入pod setup
pod update

五、通过#import “” 导入设置

在buid setting -> user header searth path -> ${SRCROOT} 为recursive

Copy详解

2015-02-01

@property (copy) NSMutableArray *array;
此写法存在两个问题:
1.添加,删除,修改数组内的元素的时候,程序会因为找不到对应的方法而崩溃.因为 copy 就是复制一个不可变 NSArray 的对象。
2.使用了 atomic 属性会严重影响性能。

nonatomic与atomic:
nonatomic即非原子属性,不会为setter方法加锁,非线程安全,适合内存小的移动设备。
atomic即原子属性,为setter方法加锁,线程安全,但需要消耗大量CPU的资源。

互斥锁(线程同步技术):
锁定1份代码只能用一把锁,用多把锁是无效的。
优点:能有效防止因多线程抢夺资源造成的数据安全问题。
缺点:需要消耗大量的CPU资源。

在非集合类对象中：对immutable对象进行copy操作,是指针复制,mutableCopy是内容复制。
对mutable对象进行copy和mutableCopy都是内容复制。

浅复制: [immutableObject copy]
深复制: [immutableObject mutableCopy]
[mutableObject copy]
[mutableObject mutableCopy]
