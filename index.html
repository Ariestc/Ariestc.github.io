JavaScriptCore使用案例及Native与JavaScript交互之内部结构剖析

2017-10-20

  1. JSVirtualMachine
    JavaScript代码是在创建JSVirtualMachine类的虚拟机下执行,不能在两个不同JSVirtualMachine类之间进行数据传输,每个JSVirtualMachine类实例有自己的堆和垃圾回收器.

  2. JSContext
    一个JSContext对象代表一个JavaScript代码执行环境,是一个全局对象,它在web开发中等同于一个window对象,同一个JSVirtualMachine类下不同的JSContext之间可以自由传值。

  3. JSValue
    是一个主要的数据类型,能够代表任何可能的JavaScript的值,一个JSValue的实例绑定到它所在的JSContext对象中。

  1. JSExport
    一种协议提供了声明式的方法去向JavaScript代码导出Objective-C的实例类及其实例方法,类方法和属性

使用PHP后端语言进行文件上传，拷贝文件的方法move_uploaded_file无权限问题解决方案

2017-06-15

1.打开终端查看httpd的owner,只有owner才有写权限:
ps aux | grep httpd

如:

_www 725 0.0 0.1 2489180 5632 ?? S 4:49下午 0:00.04 /usr/sbin/httpd -D FOREGROUND
_www 402 0.0 0.1 2488668 4836 ?? S 4:29下午 0:00.06 /usr/sbin/httpd -D FOREGROUND

2.把上传文件夹的owner改为httpd的owner

sudo chown _www /Library/WebServer/Documents/myphp/uploads

修改前:
drwxr-xr-x 2 wangl wheel 68 12 16 17:23 uploads

修改后:
drwxr-xr-x 2 _www wheel 68 12 16 17:23 uploads

3.开启文件夹owner的写权限
sudo chmod -R 0755 /Library/WebServer/Documents/myphp/uploads

如：
-rwxr-xr-x (755) 只有所有者才有读，写，执行的权限，组群和其他人只有读和执行的权限

常见权限代号:
-rw——- (600) 只有所有者才有读和写的权限
-rw-r–r– (644) 只有所有者才有读和写的权限，组群和其他人只有读的权限
-rwx—— (700) 只有所有者才有读，写，执行的权限
-rwxr-xr-x (755) 只有所有者才有读，写，执行的权限，组群和其他人只有读和执行的权限
-rwx–x–x (711) 只有所有者才有读，写，执行的权限，组群和其他人只有执行的权限
-rw-rw-rw- (666) 每个人都有读写的权限
-rwxrwxrwx (777) 每个人都有读写和执行的权限

Mac下安装PHP环境配置php+mysql+Apache+phpmyadmin及工具PhpStrom环境配置

2017-06-10

Mac默认已安装Apache和PHP环境,只需开启即可

一、开启Apache
sudo apachectl start
附:
sudo apachectl stop（关闭）
sudo apachectl -k restart(重启)
开启后浏览器输入: http://localhost 回车会显示”It works!” 表示Apache开启成功

二、开启PHP
1.修改Apache配置文件
1> 打开终端输入命令: sudo vim /etc/apache2/httpd.conf
2> 找到#LoadModule php5_module libexec/apache2/libphp5.so删除本行前面”#”字符即可
3> mac下Apache默认文件夹路劲为: /Library/WebServer/Documents 在该目录下创建一个test.php文件，文件中输入如下内容: <?php phpInfo(); ?>
浏览器输入:http://localhost/test.php 开是否显示PHPinfo页面
4> 若不成功 打开终端，输入命令：
sudo vim /etc/apache2/httpd.conf
找到①DocumentRoot “/Library/WebServer/Documents”
②DocumentRoot “/Library/WebServer/Documents”

将这两出””中的路径修改为test.php存放的路径,删除前面的”#”;
5>浏览器输入localhost/test.php,出现PHP的info页,则表示成功。

三、安装MySQL
1> 官网下载: https://dev.mysql.com/downloads/mysql/
2> 安装，注意安装完后会弹出对话框，务必保存临时密码到文件夹，以备后用
3> 启动MySQL
系统偏好设置-> MySQL
4> 点击Start MySQL Server 启动
5> 打开终端，输入命令：
/usr/local/mysql/bin/mysqladmin -u root -p’填写之前保存的临时密码’ password ‘新设置的密码’

四、安装phpmyadmin
1> 下载地址: https://www.phpmyadmin.net
下载后更改文件夹名为: phpmyadmin 并放在/Library/WebServer/Documents/目录下
2> 终端进入 cd /etc/apache2
3> ls 查看文件（输入nl httpd.conf 可给此文件加上行号）
4> vim httpd.conf（修改此文件）此步骤待确定
5> 浏览器输入: localhost/phpMyAdmin 用户名填写: root 密码填写: 新设置的密码 登陆成功即可。

五、PhpStrom环境配置
1> Mac OS X 中没有默认的php.ini文件,有对应的模版文件php.ini.default
自行拷贝修改终端输入:
sudo cp /private/etc/php.ini.default /private/etc/php.ini
2> 在Apache目录下创建PHP路径
如: /Library/WebServer/Documents/myphp
myphp为Apache路径下自定义创建的文件夹，用于放置php文件
3> 修改端口号，phpstrom默认端口号为63342，而Apache端口号为80
Preferences->Buid,Execution
添加project设置
Name为:Apache
Type为In place
Web sever root URL:http://localhost
点击apply 然后点击ok
4>点击Mappings选项
设置localpath为: /Library/WebServer/Documents/
点击apply 然后点击ok

Mac下用Homebrew进行Php、Mysql、Nginx环境安装及配置

2017-06-03

一、Homebrew安装

1.打开终端输入
ruby -e “$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)”

二、安装php-fpm
1.
1> brew tap homebrew/dupes
2> brew tap homebrew/php
3> brew install –without-apache –with-fpm –with-mysql php56
2.安装完成后将php－fpm加入环境变量中
两种方法选其一bash_profile与zshrc（注意选了后面要保持一致）
1> echo ‘export PATH=”/usr/local/sbin:$PATH”‘ >> ~/.bash_profile
2> echo ‘export PATH=”/usr/local/sbin:$PATH”‘ >> ~/.zshrc
3.创建文件夹并启动服务
1> mkdir -p ~/Library/LaunchAgents
2> ln -sfv /usr/local/opt/php56/homebrew.mxcl.php56.plist ~/Library/LaunchAgents/
3> launchctl load -w ~/Library/LaunchAgents/homebrew.mxcl.php56.plist

4.不出错的话终端输入:
lsof -Pni4 | grep LISTEN | grep php
会显示:
php-fpm 7311 wangl 6u IPv4 0x1aab2bed95440ac5 0t0 TCP 127.0.0.1:9000 (LISTEN)
php-fpm 7312 wangl 0u IPv4 0x1aab2bed95440ac5 0t0 TCP 127.0.0.1:9000 (LISTEN)
php-fpm 7313 wangl 0u IPv4 0x1aab2bed95440ac5 0t0 TCP 127.0.0.1:9000 (LISTEN)
php-fpm 7314 wangl 0u IPv4 0x1aab2bed95440ac5 0t0 TCP 127.0.0.1:9000 (LISTEN)

三、mysql 安装

1.
1> brew install mysql
2> ln -sfv /usr/local/opt/mysql/*.plist ~/Library/LaunchAgents

2.添加mysql到环境变量中
launchctl load ~/Library/LaunchAgents/homebrew.mxcl.mysql.plist

3.进入mysql的初始化环节
1> mysql_secure_installation
2> VALIDATE PASSWORD PLUGIN can be used to test passwords
and improve security. It checks the strength of password
and allows the users to set only those passwords which are
secure enough. Would you like to setup VALIDATE PASSWORD plugin?

Press y|Y for Yes, any other key for No:
(回车就行)
3> Remove anonymous users? (Press y|Y for Yes, any other key for No) :（输入y）
4> Disallow root login remotely? (Press y|Y for Yes, any other key for No) :（输入y）
5> Remove test database and access to it? (Press y|Y for Yes, any other key for No) : (输入y)
6> Reload privilege tables now? (Press y|Y for Yes, any other key for No) :(输入y)

4.测试mysql
mysql -uroot -p

(输入权限密码显示)

(退出mysql 输入: \q 可输入\h查看命令行帮助)

四、php-fpm安装及配置
1.此步骤需要翻墙
curl -L https://gist.github.com/frdmn/7853158/raw/php-fpm -o /usr/local/etc/nginx/conf.d/php-fpm

  1. 创建默认虚拟主机
    1> curl -L https://gist.github.com/frdmn/7853158/raw/sites-available_default -o /usr/local/etc/nginx/sites-available/default
    2> curl -L https://gist.github.com/frdmn/7853158/raw/sites-available_default-ssl -o /usr/local/etc/nginx/sites-available/default-ssl
    3> curl -L https://gist.github.com/frdmn/7853158/raw/sites-available_phpmyadmin -o /usr/local/etc/nginx/sites-available/phpmyadmin
  1. 使用git 来克隆实例主机 然后删除/var/www/.get文件夹 防止你的文件被git上传
    1> sudo git clone http://git.frd.mn/frdmn/nginx-virtual-host.git /var/www
    2> sudo rm -rf /var/www/.git

五、nginx的安装

1.
brew install nginx
2.确保80端口是开启
1> sudo cp -v /usr/local/opt/nginx/*.plist /Library/LaunchDaemons/
2> sudo chown root:wheel /Library/LaunchDaemons/homebrew.mxcl.nginx.plist
3.第一次开启nginx
sudo launchctl load /Library/LaunchDaemons/homebrew.mxcl.nginx.plist
附: 停止nginx
sudo launchctl unload /Library/LaunchDaemons/homebrew.mxcl.nginx.plist
4.默认的配置设置是将监听8080
curl -IL http://127.0.0.1:8080
终端显示:
HTTP/1.1 200 OK
Server: nginx/1.12.2
Date: Thu, 16 Nov 2017 01:48:05 GMT
Content-Type: text/html
Content-Length: 612
Last-Modified: Thu, 16 Nov 2017 01:45:44 GMT
Connection: keep-alive

Accept-Ranges: bytes
六、安装ssl
1.创建文件夹来存放ssl的证书和私钥
mkdir -p /usr/local/etc/nginx/ssl
2.生成4096比特 的RSA秘钥
1> openssl req -new -newkey rsa:4096 -days 365 -nodes -x509 -subj “/C=US/ST=State/L=Town/O=Office/CN=localhost” -keyout /usr/local/etc/nginx/ssl/localhost.key -out /usr/local/etc/nginx/ssl/localhost.crt
2> openssl req -new -newkey rsa:4096 -days 365 -nodes -x509 -subj “/C=US/ST=State/L=Town/O=Office/CN=phpmyadmin” -keyout /usr/local/etc/nginx/ssl/phpmyadmin.key -out /usr/local/etc/nginx/ssl/phpmyadmin.crt

3.使用虚拟主机启用文件夹
1> ln -sfv /usr/local/etc/nginx/sites-available/default /usr/local/etc/nginx/sites-enabled/default
2> ln -sfv /usr/local/etc/nginx/sites-available/default-ssl /usr/local/etc/nginx/sites-enabled/default-ssl
3> ln -sfv /usr/local/etc/nginx/sites-available/phpmyadmin /usr/local/etc/nginx/sites-enabled/phpmyadmin
4.打开nginx
sudo launchctl load /Library/LaunchDaemons/homebrew.mxcl.nginx.plist

七、对服务的控制

  1. 重新启动来进行重命名
    1>curl -L https://gist.github.com/frdmn/7853158/raw/bash_aliases -o /tmp/bash_aliases
    2> cat /tmp/bash_aliases >> ~/bash_aliases
    3> echo “source ~/bash_aliases” >> ~/bash_profile

2.更新我们的变量 让设置生效
source ~/bash_profile

Run Loop

2017-04-23

1.什么是runloop
1>字面意思: 运行循环
2>内部是do-while循环 在这个循环内部不断的处理各种任务(如:source、timer、observer)
3>一个线程对应一个runloop,主线程的runloop默认已经启动，子线程的runloop需要手动启动 即调用run方法
4>runloop只能选择一个mode启动，如果当前mode中没有任何source、Timer、observer，那么就直接退出runloop

2.自动释放池什么时候释放?
在runloop 睡眠之前kCFRunLoopBeforeWaiting

3.在开发中如何使用runloop? 什么应用场景？
1>开启一个常驻线程 即：让一个子线程不进入消亡状态，等待其他线程发来消息,处理其他事件
2>在子线程中开启一个定时器或监控
3>可以控制某些事件(行为、任务)或定时器在特定模式下执行
4>可以添加observer监听runloop的状态，比如监听点击事件的处理(在所有点击事件之前做些事情)
